# 韭研产业研究图片样例特征

## 样例范围

本目录保存韭研产业研究 OCR 参考图片，用于约束 `jiuyan__industry_image_ocr_results` 的 prompt 和 JSON schema。

当前样例：

- `2144539A-6408-4DD5-B951-2D1D92EC6BE6.png`：培育钻石/钻石散热主题。
- `6A0630EE-0B2E-4805-AAE3-79655BF587EC.png`：AI PCB 主题。

## 图片结构特点

- 图片主体是长表格，左侧通常是合并单元格形式的主题分类，右侧是一行一个股票或公司条目。
- 表头不稳定。固定出现概率较高的是 `个股`；说明列可能叫 `相关性`、`AI服务器相关`、`2024年相关营收占比` 或其它业务表头；来源列可能叫 `信源`、`资料来源`、`来源`。
- 左侧主题可能是多层路径。例如 `AI服务器相关 > 覆铜板`，或 `钻石散热 > 金刚石热片`。
- 同一个主题合并单元格会覆盖多行，OCR 需要把该主题路径带到每个股票条目上。
- 部分股票名称会用红色标记，表示图或更新节点里涨幅较大；颜色不是抽取字段的必要条件。
- 表格底部可能有风险提示、说明文字、跳转链接和水印，这些不是股票条目，应忽略。
- 有些行是交易所、指数或大厂映射，例如 `北交所`、`谷歌`、`AWS`、`Meta`，只有在表格把它们放在 `个股` 列且像条目行一样给出证据时才抽取。

## 抽取原则

- 不依赖固定表头名识别说明列，除 `个股` 和来源列外，其它业务列统一作为证据单元格保存。
- 每条输出代表一个股票/公司条目，不代表一个主题合并单元格。
- `theme_path` 保存左侧合并分类和最相关的主题层级，按从大到小顺序排列。
- `topic` 保存最外层最大主题，通常取 `theme_path` 第一项；如果没有主题路径，则为空字符串。
- `relevance` 保存该行最核心的相关性说明，可从动态业务列中综合选择，但不要编造。
- `evidence_cells` 保存表格中除 `个股` 外的主要可见单元格，必须保留原始表头和文字。
- `source` 保存来源列文字，例如 `互动`、`公告`、`调研`、`财报`、`研报`、`网传纪要`。

]
