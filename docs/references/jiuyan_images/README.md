## 图片结构特点

- 图片主体是长表格，左侧通常是合并单元格形式的主题分类，右侧是一行一个股票或公司条目。
- 表头不稳定。固定出现概率较高的是 `个股`；说明列可能叫 `相关性`、`AI服务器相关`、`2024年相关营收占比` 或其它业务表头；来源列可能叫 `信源`、`资料来源`、`来源`。
- 左侧主题可能是多层路径。例如 `AI服务器相关 > 覆铜板`，或 `钻石散热 > 金刚石热片`。
- 同一个主题合并单元格会覆盖多行，OCR 需要把该主题路径带到每个股票条目上。
- 部分股票名称会用红色标记，表示图或更新节点里涨幅较大；颜色不是抽取字段的必要条件。
- 表格底部可能有风险提示、说明文字、跳转链接和水印，这些不是股票条目，应忽略。
- 有些行是交易所、指数或大厂映射，例如 `北交所`、`谷歌`、`AWS`、`Meta`，只有在表格把它们放在 `个股` 列且像条目行一样给出证据时才抽取。
