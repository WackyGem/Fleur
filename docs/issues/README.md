# Issues

本目录合并原 `docs/debt/` 和 `docs/optimize/`，用于追溯技术债、实现漂移、质量扫描、维护性审计和治理建议。

## 使用规则

- 新发现的问题优先进入对应架构事实文档、RFC、plan 或 job report；不要在本目录新增活跃待办。
- 已关闭、已转为计划或只作为历史证据的问题记录放入 [archive/](archive/)。
- 技术债和优化建议不作为当前架构约束；长期规则进入 `docs/ADR/` 或 `docs/architecture/`。

## 归档入口

| 目录 | 用途 |
|---|---|
| [archive/debt/](archive/debt/) | 已关闭技术债和实现漂移记录 |
| [archive/optimize/](archive/optimize/) | 质量扫描、维护性审计和治理建议 |
