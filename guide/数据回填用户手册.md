# 数据回填用户手册

## 适用版本

- scheduler: `0.1.0`
- Dagster Web UI job: `backfill__fetch_history_sources_to_marts_job`
- 适用场景：S3 source 数据和 ClickHouse raw 数据都为空，需要初始化 source -> raw -> stg -> int -> calculation -> mart。

## 执行策略

本次改为两阶段执行：

| 阶段 | execution_mode | 目的 |
| --- | --- | --- |
| 阶段一 | `source_raw_only` | 先把 S3 source 和 ClickHouse raw 全部回填完成 |
| 阶段二 | `downstream_only` | 一次性从 raw 推进到 stg、int、calculation 和 mart |

阶段一不要用 `all_source_to_marts` 一次性执行。各数据域的历史起点不同，统一日期窗口会扩大不必要的 source/raw 回填范围。

阶段二可以用 `all_source_to_marts` 一次性执行完整 downstream。此时不会重新跑 source/raw；`start_date` / `end_date` 主要用于 Furnace 技术指标的历史修复窗口，dbt stg/int/mart 会基于阶段一已经写入的 raw 数据重建。

## 本次回填范围

| 顺序 | target_scope | 日期范围 | 说明 |
| --- | --- | --- | --- |
| 1 | `snapshot_reference_data` | 日期必填但会被忽略 | 交易日历、股票基础信息及其 stg/int/mart |
| 2 | `baostock_daily_kline` | `2021-01-01` 到 2026 年当前可回填截止日 | BaoStock 日 K、股票行情 int、6 个 Furnace calculation、技术指标 marts |
| 3 | `market_events` | `2025-01-01` 到 2026 年当前可回填截止日 | 当前 source-to-marts 中只包含 THS limit up pool；Jiuyan 异动排除 |
| 4 | `eastmoney_f10` | `1990-01-01` 到 2026 年当前可回填截止日 | EastMoney F10、股本/除权/估值/行情相关 int 和 mart |
| 5 | `chinabond` | `2006-01-01` 到 2026 年当前可回填截止日 | ChinaBond 国债收益率、risk free rate int 和 mart |

本手册示例使用 `2026-07-01` 作为 2026 年当前可回填截止日。实际执行时替换成运行当天或更早的有效日期；不要提前填 `2026-12-31`。

## Web UI 入口

在 Dagster Web UI 中进入：

```text
Jobs -> backfill__fetch_history_sources_to_marts_job -> Launchpad
```

每个配置都按同样流程执行：

1. 先粘贴配置并保持 `dry_run: true`。
2. 启动 dry-run。
3. 在日志中确认出现 `Backfill source-to-marts plan`，并确认 step 范围正确。
4. 回到 Launchpad，把同一份配置的 `dry_run` 改成 `false`。
5. 启动真实回填。

阶段二必须等阶段一所有真实回填成功后再执行。

## 阶段一：S3 Source -> ClickHouse Raw

阶段一全部使用：

```yaml
execution_mode: source_raw_only
```

### 1. Snapshot Reference Data

```yaml
ops:
  backfill__fetch_history_sources_to_marts_controller:
    config:
      target_scope: snapshot_reference_data
      start_date: "2026-07-01"
      end_date: "2026-07-01"
      execution_mode: source_raw_only
      refresh_prerequisite_snapshots: false
      overwrite_source_partitions: false
      dry_run: true
```

确认 dry-run 只包含 `source_raw`，并且包含：

- `source/sina__trade_calendar`
- `source/baostock__query_stock_basic`
- `clickhouse/raw/sina__trade_calendar`
- `clickhouse/raw/baostock__query_stock_basic`

### 2. BaoStock 日 K

```yaml
ops:
  backfill__fetch_history_sources_to_marts_controller:
    config:
      target_scope: baostock_daily_kline
      start_date: "2021-01-01"
      end_date: "2026-07-01"
      execution_mode: source_raw_only
      refresh_prerequisite_snapshots: false
      overwrite_source_partitions: false
      dry_run: true
```

确认 dry-run 按年份展开：

```text
baostock daily source YYYY
baostock compacted source YYYY
baostock raw YYYY
```

### 3. Market Events

```yaml
ops:
  backfill__fetch_history_sources_to_marts_controller:
    config:

      end_date: "2026-07-01"

      refresh_prerequisite_snapshots: false
      overwrite_source_partitions: false

```

确认 dry-run 只包含 THS 链路：

- `source/ths__limit_up_pool`
- `source/ths__limit_up_pool_compacted`
- `clickhouse/raw/ths__limit_up_pool_compacted`

当前 source-to-marts 入口会过滤 Jiuyan 异动，不会把 `source/jiuyan__action_field` 或对应 raw 纳入本次回填。

## 阶段二：一次性 Stg -> Int -> Calculation -> Mart

阶段二只执行一次 Web UI run，使用：

```yaml
execution_mode: downstream_only
target_scope: all_source_to_marts
```

阶段二不会重新跑 source/raw。先 dry-run，再把 `dry_run` 改成 `false` 执行真实 downstream。

这里的 `start_date` 使用 `2021-01-01`，因为当前需要 Furnace 股票技术指标覆盖 BaoStock 日 K 的回填范围。EastMoney F10 的 `1990-2026` 和 ChinaBond 的 `2006-2026` 已经在阶段一写入 raw；dbt downstream 会从 raw 表重建，不依赖这个日期窗口裁剪。

```yaml
ops:
  backfill__fetch_history_sources_to_marts_controller:
    config:
      target_scope: all_source_to_marts
      start_date: "2021-01-01"
      end_date: "2026-07-01"
      execution_mode: downstream_only
      refresh_prerequisite_snapshots: false
      overwrite_source_partitions: false
      dry_run: true
```

确认 dry-run 不包含 `source_raw`，并包含这些 downstream 阶段：

- `dbt_staging`
- `dbt_intermediate`
- `furnace_calculation`
- `dbt_calculation_wrappers`
- `dbt_marts`

确认覆盖范围包括：

- Snapshot reference data downstream：交易日历、股票基础信息、基础 snapshot int/mart。
- BaoStock downstream：股票行情核心 int、Furnace calculation、calculation wrappers、股票行情和技术指标 marts。
- Market events downstream：`stg_ths__limit_up_pool_compacted`。
- EastMoney F10 downstream：9 个 `stg_eastmoney__*`、股本/除权/估值/行情相关 int、`mart_stock_quotes_daily`。
- ChinaBond downstream：`stg_chinabond__government_bond`、`int_government_bond_yields_daily`、`int_risk_free_rate_daily`、`mart_risk_free_rate_daily`。

`furnace_calculation` 应包含 6 个股票技术指标：

- `fleur_calculation/calc_stock_kdj_daily`
- `fleur_calculation/calc_stock_ma_daily`
- `fleur_calculation/calc_stock_rsi_daily`
- `fleur_calculation/calc_stock_boll_daily`
- `fleur_calculation/calc_stock_macd_daily`
- `fleur_calculation/calc_stock_price_pattern_daily`

dry-run 时 Furnace child config 使用 `mode: dry-run`；真实执行时使用历史修复语义 `mode: replace-cascade`。

## 失败恢复

### 阶段一失败

优先修复失败原因后，重新执行同一个 target_scope 的阶段一配置。已经成功的其他 scope 不需要重跑。

如果 BaoStock source 分区已经写出但需要重新抓取，才把：

```yaml
overwrite_source_partitions: true
```

空库首次回填保持 `false`。

### 阶段二失败

修复失败原因后，重新执行阶段二配置，也就是继续使用：

```yaml
target_scope: all_source_to_marts
execution_mode: downstream_only
```

阶段二不会重新提交 source/raw。

### 日期写到未来导致失败

如果日志出现类似：

```text
end_date cannot be later than today's Asia/Shanghai date
```

把 `end_date` 改成运行当天或更早的有效日期后重试。

## 验收方式

在 Dagster Web UI 的 Runs 页面按 tags 搜索或过滤：

```text
backfill.kind=fetch_history_sources_to_marts
```

阶段一至少确认这些真实 parent run 成功：

- `backfill.target_scope=snapshot_reference_data`
- `backfill.target_scope=baostock_daily_kline`
- `backfill.target_scope=market_events`
- `backfill.target_scope=eastmoney_f10`
- `backfill.target_scope=chinabond`

阶段二确认 1 个真实 parent run 成功：

- `backfill.target_scope=all_source_to_marts`

每个 parent run 的 run config 或日志中应能看到对应的 `execution_mode`。真实回填时，失败会停在当前失败 step，不会继续提交后续 stage。
